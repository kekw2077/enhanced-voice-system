# EVS — Enhanced Voice System

**EVS** — десктопный голосовой ассистент для **Windows**: постоянное прослушивание со
словом-активатором, выполнение команд, распознавание и синтез речи, чат с ИИ
(локально или через удалённый сервер) и плавающий виджет с живой визуализацией
голоса. Работает офлайн с локальными моделями или подключается к удалённому LLM.

> Только Windows. Установщик компактный (~16 МБ), тяжёлые компоненты (голосовой
> движок, клонирование голоса) догружаются по требованию. Автообновления — через
> собственный подписанный канал.

## Возможности

### Голос
- **Слово-активатор** «EVS» (распознаётся и в русской, и в английской речи),
  постоянное прослушивание, озвучка ответов — работа «как у голосового ассистента».
- **Распознавание речи (STT):** Whisper (мультиязычный, выбор размера модели) и
  GigaAM (русский). Движок распознавания ставится отдельным компонентом при
  первом включении голоса.
- **Синтез речи (TTS):** голоса piper, **клонирование голоса** (ответы вашим
  голосом из образца WAV на 6–10 секунд, офлайн), а также CosyVoice — при
  подключении соответствующего сервера.
- **Каталог голосовых команд:** открытие приложений, ярлыков (`.lnk`), файлов и
  ссылок, управление медиа и громкостью; видимый отклик (что услышано, статус
  движка, уведомления о выполнении).

### ИИ-чат
- **Локальный режим** — инференс прямо на устройстве через
  [fllama](https://github.com/Telosnex/fllama) (llama.cpp, GGUF-модели), полностью
  офлайн. Встроенный каталог моделей разбит на тиры по мощности устройства.
- **Удалённый режим** — подключение к любому Ollama- или OpenAI-совместимому API
  (свой сервер, локальная сеть и т. п.).
- Несколько диалогов с историей, поиском и закреплением; персонализация ассистента
  (тон, формальность, многословность и т. п.) и память/профиль «о вас».

### Десктоп-оболочка
- **Плавающий виджет** — отдельное всегда-поверх окно с живой визуализацией голоса
  (волна/сфера/кольцо/полосы/частицы), реагирующей на речь и озвучку.
- **Системный трей**, глобальные горячие клавиши, автозапуск при входе в систему,
  собственная безрамочная тема окна.
- **Удалённый ввод с телефона** — отправка команд/текста в EVS по локальной сети.
- Метрики системы (CPU/RAM/VRAM) в боковой панели.
- Темы оформления (Claude — светлая/тёмная, нейтральная тёмная), интерфейс RU/EN.

### Обновления
- Тонкие обновления: компактный установщик + догрузка тяжёлых компонентов по
  требованию. Автообновления через собственный канал (appcast + **подписанные**
  установщики), окно «Что нового» со списком изменений.

## Системные требования

Требования зависят от режима: удалённый LLM нетребователен к железу, локальные
модели и офлайн-распознавание требуют больше памяти и (желательно) видеокарты.

### Минимальные
- **ОС:** Windows 10 64-bit (версия 1809 или новее) или Windows 11.
- **CPU:** двухъядерный x64.
- **ОЗУ:** 4 ГБ (удалённый режим, лёгкая работа).
- **Диск:** ~500 МБ на приложение + ~1–2 ГБ на голосовой движок (догружается при
  включении голоса); плюс место под локальные модели, если используете их.
- **Микрофон** — для голосового ввода и слова-активатора.
- **Интернет** — для удалённых моделей и первичной загрузки компонентов/моделей
  (в чистом офлайне после загрузки не обязателен).

### Рекомендуемые
- **ОС:** Windows 11 64-bit.
- **CPU:** современный 4+ ядерный x64.
- **GPU:** дискретная видеокарта — заметно ускоряет локальный инференс и офлайн-
  распознавание (не обязательна).
- **ОЗУ:** 16 ГБ. Локальные LLM 7–8B в квантовании Q4 занимают дополнительно
  ~5–6 ГБ; больше памяти — крупнее модель и контекст.
- **Диск:** SSD; запас под модели (GGUF-модели — от ~0.5 до ~5 ГБ каждая).
- Отдельный микрофон/гарнитура — для стабильного распознавания.

> Полностью офлайн-работа возможна (локальная модель + офлайн STT/TTS), но требует
> места под модели и больше RAM/GPU. Удалённый режим (Ollama / OpenAI-совместимый
> сервер) минимально требователен к железу. GPU ускоряет, но не обязателен.

## Установка и обновления

1. Скачайте установщик `EVS-Setup-x.y.z.exe` со страницы
   [Releases](https://github.com/kekw2077/enhanced-voice-system/releases) и
   запустите его.
2. При первом включении голосовых функций EVS догрузит голосовой движок.
3. Дальше приложение обновляется само: новые версии приходят в фоне, появляется
   плашка/окно «Что нового» — обновление применяется в один клик.

## Технологии

Flutter (Windows desktop), прямые вызовы Win32 (трей, горячие клавиши, управление
окном, single-instance), локальный инференс через fllama (llama.cpp), офлайн
распознавание/синтез речи через отдельный голосовой компонент, автообновление в
стиле Sparkle (WinSparkle / `auto_updater`) с подписанными установщиками.

Проект — одна кодовая база; Flutter-приложение находится в подпапке
[`test1/`](test1/). Подробности архитектуры — в [`test1/CLAUDE.md`](test1/CLAUDE.md).

## Статус

Персональный проект. EVS — самостоятельное десктоп-приложение (ранее
разрабатывалось в рамках проекта Mirai; здесь ведётся как отдельный продукт,
только для Windows).
