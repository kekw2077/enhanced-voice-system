# Хендофф: доделать светлую тему **Apple** (сессия на ПК с Flutter)

Этот файл — инструкция для сессии Claude Code **на компьютере пользователя**, где
есть Flutter (`flutter run -d windows` / `flutter analyze`) и можно видеть UI.
Ветка: **`desktop`**. Файл: почти весь код в `test1/lib/main.dart`.

> **Задача:** довести тему **Apple (светлую)** до состояния «ничего не сломано
> визуально» — убрать невидимый текст и тёмные плашки на светлом фоне. Тему
> **не переписывать** — она уже заведена; нужно точечно перекрасить оставшиеся
> зашитые «под тёмную» цвета. Метод — **вариант А: по факту** (запустить, смотреть
> глазами, править конкретные места), НЕ слепая массовая замена.

## 2. Инфраструктура для правок — добавить недостающие токены и хелперы

Часть семантических токенов при первичной сборке была намеренно урезана, чтобы не
плодить неиспользуемые символы. Перед правками **верни их** (иначе не на что
маппить). В `class _Palette` добавь поля и проставь во всех трёх инстансах:

| поле | смысл | dark | steam | discord | **apple** | **claude** |
|---|---|---|---|---|---|---|
| `body` | светло-серый «телесный» текст | `#D0D4E2` | `#C6D4DF` | `#DBDEE1` | `#1D1D1F` | `#3D3D3A` |
| `faint` | приглушённый/третичный текст | `#6E7280` | `#66707A` | `#949BA4` | `#86868B` | `#8E8B82` |
| `success` | «готово/успех» | `#54E08A` | `#8FD14F` | `#23A55A` | `#1E8E3E` | `#5DB872` |
| `danger` | ошибка | `#E05D5D` | `#FF6B6B` | `#ED4245` | `#D93025` | `#C64545` |

`card2`, `accent`, `stroke` уже есть в `_Palette`. Затем добавь рядом со швами
(`_bg`/`_card`/…) хелперы:

```dart
Color _body(BuildContext c) => _pal(c).body;
Color _faint(BuildContext c) => _pal(c).faint;
Color _accent(BuildContext c) => _pal(c).accent;
Color _stroke(BuildContext c) => _pal(c).stroke;
Color _card2(BuildContext c) => _pal(c).card2;
Color _success(BuildContext c) => _pal(c).success;
Color _danger(BuildContext c) => _pal(c).danger;
```
> Каждый добавленный хелпер должен где-то использоваться, иначе `flutter analyze`
> выдаст `unused_element`. Добавляй хелпер вместе с первой его заменой.

## 4. КАК менять безопасно (ключевые подводные камни)

Именно из-за них нельзя было делать это слепой массовой заменой в оффлайн-сессии.

1. **`const`-каскад.** Замена `Color(0xFF…)` на `_body(context)` делает значение
   **рантайм**, значит нужно снять `const` с этого узла:
   - `const TextStyle(color: Color(0xFFD0D4E2))` → `TextStyle(color: _body(context))`.
   - Если `const` был выше (`const Text('x', style: TextStyle(...))`) — снять `const`
     и там (до ближайшей границы). `flutter analyze` покажет
     `const_with_non_constant_argument` / `invalid_constant` — правь по цепочке вверх.
2. **`context` в области видимости.** Хелперы требуют `BuildContext`. Имя локальной
   переменной в этом файле бывает `context`, `c`, `ctx`, `dctx`. Подставляй **то
   имя, что реально в области** данного `build`/`builder`. Если контекста нет
   (top-level `const`, `static const`, поля класса) — хелпер там применить нельзя:
   вынеси стиль в `build`, либо задай цвет через `Theme.of(context)` в точке
   использования.
3. **`Colors.white` — разбирать по месту.** Оставляй белым, если это:
   - иконка/текст **на цветной кнопке** (синий/градиент) — там белый корректен на
     всех темах;
   - контент поверх тёмного медиа.
   Меняй на `_txt(context)`, если это текст **на карточке/фоне** (на Apple фон
   белый → белый текст исчезает).
4. **Проверяй все три темы после каждой партии.** Правка не должна испортить Dark
   или Steam. Все токены уже заданы для трёх тем, поэтому один `_body(context)`
   работает во всех — но глазами проверь Dark/Steam тоже.
5. **Мелкими партиями.** 1 экран/раздел → `flutter analyze` → визуальная проверка
   Apple + Dark → коммит. Так ошибки локальны и откат дешёвый.

## 5. Порядок работы (рекомендуемый)

1. Добавить токены/хелперы (п.2).
2. Взять первый экран из списка сломанных (п.1). Заменить его зашитые цвета по
   таблице (п.3), соблюдая п.4.
3. `flutter analyze` — 0 ошибок. Запустить, переключить Apple/Dark/Steam — глазами.
4. Коммит на `desktop` (маленький, по экрану/разделу):
   ```bash
   git add -A && git commit -m "EVS Apple theme: tokenize colors on <screen>"
   git push origin desktop
   ```
5. Повторять по списку, пока Apple не станет чистой на всех экранах.
